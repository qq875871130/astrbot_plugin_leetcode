# LeetCode 每日一题插件

🎯 功能强大的 LeetCode 每日一题插件，支持多语言显示、个人订阅、AI解题分析、定时推送等功能。

## ✨ 功能特性

- 📅 **每日一题推送** - 定时推送 LeetCode 每日一题到订阅群组
- 👤 **个人订阅** - 支持用户私信订阅，独立管理个人推送偏好
- 🌐 **多语言显示** - 支持中文/英文/双语题目显示
- 🔍 **题目查询** - 支持按题号查询任意 LeetCode 题目
- 📝 **完整题目内容** - 显示题目描述、难度、通过率、标签等详细信息
- 🤖 **AI解题分析** - 接入 AstrBot LLM，提供智能解题思路和参考代码
- 🔗 **题目链接** - 一键跳转到 LeetCode 原题页面
- 👑 **权限管理** - 群聊命令需管理员权限，私聊命令对所有用户开放
- 💾 **本地缓存** - 缓存今日题目，避免重复请求
- ⚙️ **灵活配置** - 支持自定义推送时间、语言偏好等参数

## 🚀 安装

### 方式一：通过 AstrBot 插件市场安装

1. 打开 AstrBot 管理面板
2. 进入「插件」→「插件市场」
3. 搜索 `LeetCode` 并安装

### 方式二：手动安装

1. 克隆仓库到插件目录：
   ```bash
   cd /path/to/astrbot/data/plugins
   git clone https://github.com/NumInvis/astrbot_plugin_leetcode.git
   ```

2. 重启 AstrBot

## ⚙️ 配置

在 AstrBot 插件配置面板中设置：

| 配置项 | 类型 | 说明 | 默认值 |
|--------|------|------|--------|
| `admin_users` | 列表 | 管理员用户ID列表 | `[]` |
| `default_language` | 字符串 | 默认题目语言（zh/en/both） | `zh` |
| `enable_personal_subscribe` | 布尔 | 启用个人订阅功能 | `true` |
| `personal_inform_hour` | 整数 | 个人订阅推送时间（小时） | `9` |
| `personal_inform_minute` | 整数 | 个人订阅推送时间（分钟） | `30` |
| `enable_llm_translation` | 布尔 | 启用大模型翻译中文 | `true` |
| `translation_provider_id` | 字符串 | 翻译专用LLM提供商ID（可选） | `''` |
| `enable_image_push` | 布尔 | 推送时使用文转图（图片形式发送） | `false` |

### ⚠️ 重要：中文功能依赖大模型

**中文题目内容通过大模型API实时翻译**，使用前请确保：

1. **AstrBot 已配置 LLM 提供商**（如 OpenAI、Claude、Gemini 等）
2. **翻译功能默认开启**，可在配置中关闭（此时只显示英文）
3. **建议选择擅长翻译的模型**，如 GPT-4、Claude 3.5 Sonnet 等
4. **翻译过程需要一定时间**，首次获取中文题目时可能有短暂延迟

### 配置管理员

在 AstrBot 配置文件中添加管理员用户ID：

```json
{
  "admin_users": ["123456789", "987654321"]
}
```

### 语言配置说明

- `zh`: 仅显示中文题目（**需要LLM翻译**）
- `en`: 仅显示英文题目
- `both`: 双语显示（**中文部分需要LLM翻译**）

用户可以通过 `/lc语言` 命令覆盖全局默认设置，仅影响个人接收的题目显示。

### 翻译配置

```json
{
  "enable_llm_translation": true,
  "translation_provider_id": ""  // 留空使用AstrBot默认LLM提供商
}
```

**如何获取 LLM 提供商 ID：**

在 AstrBot 聊天中发送 `/provider` 指令，即可查看当前已配置的所有 LLM 提供商列表及其 ID。将对应的 ID 填入 `translation_provider_id` 即可指定专用翻译提供商。如果留空，则自动使用 AstrBot 的默认提供商。

```
/lc题目 1

```
/lc解题 1

订阅后，每天会自动推送当日题目到群内。

### 个人订阅使用指南

> ⚠️ **个人订阅需要 Bot 开启主动型能力。**
>
> 个人订阅的定时推送是 Bot **主动向用户发送消息**，而非回复用户消息。请在 AstrBot **AI 配置**页面中启用 **主动型能力**，否则 Bot 无法主动发送私信推送。

#### 👤 订阅个人推送
私信机器人发送：
```
/lc订阅我
```
订阅成功后，每天会在设定时间收到题目私信推送。

#### 🌐 设置语言偏好
```
/lc语言 zh
```
可选值：
- `zh` - 仅中文
- `en` - 仅英文
- `both` - 双语显示

#### ⏰ 设置推送时间
```
/lc时间 8:00
```
设置每天推送题目的时间，使用 24 小时制。

- `/lc时间` — 查看当前推送时间
- `/lc时间 8:00` — 设置为 08:00
- `/lc时间 22:30` — 设置为 22:30
- `/lc时间 默认` — 恢复使用配置文件中的默认时间

> 配置文件中的 `personal_inform_hour` / `personal_inform_minute` 作为所有用户的默认推送时间，每个用户可通过此命令单独覆盖。

#### 📊 查看个人状态
```
/lc我的状态
```
显示当前订阅状态和语言偏好设置。

#### ❌ 取消个人订阅
```
/lc退订我
```

## 🔌 AI解题前置要求

使用 `/lc解题` 命令需要：

1. AstrBot 已配置 LLM 提供商（如 OpenAI、Claude、Gemini 等）
2. 在 AstrBot 管理面板中配置好相应的 API Key

支持的大模型：
- OpenAI (GPT-3.5/GPT-4)
- Anthropic (Claude)
- Google (Gemini)
- 以及其他 AstrBot 支持的 LLM 提供商

## 🌐 数据来源与翻译

### 数据来源
- **每日一题数据**: [LeetCode API](https://leetcode.com/)
- **题目详情数据**: [lcid.cc](https://lcid.cc/)（元信息）+ [LeetCode GraphQL](https://leetcode.com/graphql)（完整题干和标签）

### 中文翻译说明
- **中文标题和内容**：通过大模型API（如 GPT-4、Claude 等）实时翻译
- **翻译时机**：首次获取题目时自动翻译并缓存
- **翻译质量**：取决于所使用的LLM模型，建议使用 GPT-4 或 Claude 3.5 Sonnet 以获得最佳效果
- **失败回退**：如翻译失败，自动显示英文原文

## 📝 更新日志

### v1.1.0

#### ✨ 新增特性

- **多语言显示支持** - 新增中文/英文/双语三种显示模式
  - 中文内容通过大模型 API 实时翻译
  - 新增 `/lc语言` 命令设置个人语言偏好
  - 智能链接适配 - 根据语言自动切换链接域名（中文使用 leetcode.cn，英文/双语使用 leetcode.com）
- **个人订阅系统** - 独立于群组的个人订阅功能
  - 支持私信接收每日一题推送
  - 新增 `/lc订阅我`、`/lc退订我`、`/lc我的状态` 命令
  - 新增 `/lc时间 [HH:MM]` 命令，支持每个用户自定义推送时间
- **LLM 翻译配置** - 支持自定义翻译提供商和翻译开关

#### 🔧 功能优化

- **`/lc题目` 和 `/lc解题` 题干获取重构** - 修复按题号查询时无法获取题目内容的问题。原先仅通过 `lcid.cc/info/{id}` 获取元信息（不含题干），现在在此基础上额外通过 LeetCode GraphQL API 获取完整题干描述和标签信息，与每日一题的数据丰富度一致
- **`/lc解题` 流程优化** - 有题干时先输出题目信息（与 `/lc题目` 展示格式一致），再开始 AI 分析解答，用户可先阅读题目再等待解析结果
- **通过率和标签数据完善** - 修复 `acRate` 硬编码为 0 的问题，现在从 API 正确获取；标签从 GraphQL 返回的 `topicTags` 中获取实际内容而非空列表
- **权限管理细化** - 群聊命令需管理员权限，私聊个人命令对所有用户开放
- **配置体系重构** - 新增语言设置、个人订阅开关、个人推送时间、翻译开关等配置项
- **数据持久化** - 订阅数据、用户语言偏好等动态配置采用 JSON 文件存储，支持并发写入锁
- **HTML → Markdown 转换重构** - 使用基于 HTMLParser 的专用转换器替代原有 `clean_html()` 简单清理，自动将代码块（含语言标识）、加粗、斜体、标题、列表、引用、分割线等 HTML 元素转换为标准 Markdown 格式，无需依赖大模型即可获得结构化的题目内容
- **文转图推送** - 新增 `enable_image_push` 配置项，开启后订阅推送将以图片形式发送（默认关闭，纯文本）

#### 🐛 问题修复

- **配置读取修复** - 修复了配置无法从 AstrBot 正确读取的问题
- **API 请求重试** - 为 LeetCode API 和 lcid.cc API 请求增加自动重试机制（最多 3 次，间隔 1s），应对 Vercel 免费层偶发的连接重置（WinError 10054）等问题
